# Ticket Summary Prompt

根据当前对话生成结构化工单摘要。摘要必须客观、简洁，不补充对话中不存在的信息。

## 输出格式

### 问题标题
用一句话概括客户问题，避免夸大影响范围。

### 客户环境
- 数据库、服务、系统或客户端类型。
- 版本、部署环境或访问路径。
- 未收集到的信息标记为“待补充”。

### 问题现象
- 客户看到的错误、异常或性能表现。
- 发生时间、频率和影响范围。

### 已收集信息
- 客户已经提供的关键事实。
- 已完成的检查结果。
- 明确缺失的信息。

### 初步原因分析
- 基于分类和已知信息给出可能原因。
- 使用“可能”或“初步判断”，不要写成最终结论。

### 排查步骤
- 已建议客户执行的低风险检查。
- 下一步建议收集的信息。
- 高风险操作必须标记需要人工确认。

### 当前状态
- collecting_info：仍在收集信息。
- investigating：已有初步线索，正在排查。
- pending_human：需要人工介入。
- draft：摘要草稿。

### 后续跟进事项
- 列出需要客户补充的信息。
- 列出需要技术支持人员确认的事项。
- 不记录密码、密钥、token 或真实敏感信息。
