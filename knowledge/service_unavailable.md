# 服务不可用

## 适用场景
客户反馈服务启动失败、服务不可用、进程不存在、端口未监听、服务 down 或请求无法到达后端。

## 常见问题现象
- 服务启动命令执行失败。
- 页面或客户端提示服务不可用。
- 进程不存在或端口未监听。
- 日志中出现启动异常、配置错误或依赖不可达。

## 需要向客户收集的信息
- 服务名称、版本和部署环境。
- 启动命令、启动用户和工作目录。
- 进程状态和端口监听状态。
- 最近一次启动日志或错误摘要。
- 最近发布、配置变更或依赖变更。
- 影响范围和是否有备用节点。

## 可能原因
- 配置文件错误或缺少必要环境变量。
- 端口被占用或未正确监听。
- 依赖服务不可达。
- 权限不足导致无法读取文件或绑定端口。
- 最近发布引入启动失败。

## 标准排查步骤
1. 确认服务名称、部署环境和启动方式。
2. 检查进程是否存在。
3. 检查端口是否监听。
4. 查看启动日志和最近错误摘要。
5. 对比最近发布、配置和依赖变更。
6. 判断是否需要回滚、重启或人工介入。

## 常用命令或 SQL 示例
```bash
ps -ef | grep <service-name>
netstat -tulnp | grep <port>
ss -lntp | grep <port>
systemctl status <service-name>
journalctl -u <service-name> --since "30 minutes ago"
```

## 风险操作提醒
- 重启、回滚、修改配置会影响业务，必须人工确认。
- 不要要求客户发送完整未脱敏日志。
- 涉及生产服务操作时，需要确认影响范围和维护窗口。

## 需要转人工的情况
- 服务完全不可用或影响生产业务。
- 需要执行重启、回滚或配置修改。
- 日志显示依赖故障、安全策略阻断或资源耗尽。
- 客户无法确认服务拓扑和影响范围。

## 工单摘要要点
- 服务名称、版本和部署环境。
- 启动命令、进程状态和端口监听状态。
- 日志错误摘要。
- 最近变更和影响范围。
- 已尝试操作和下一步建议。
