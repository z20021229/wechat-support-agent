# 权限问题

## 适用场景
客户反馈执行 SQL、访问对象、调用功能或登录系统时提示权限不足、permission denied、授权失败或用户无权限。

## 常见问题现象
- 当前用户执行操作时报权限不足。
- 查询表、视图、过程或文件时报 denied。
- 授权后仍提示无权限。
- 某个角色或用户在不同环境权限表现不一致。

## 需要向客户收集的信息
- 当前登录用户和角色。
- 执行的 SQL、操作路径或功能名称。
- 目标对象名称，例如库、表、视图、过程或目录。
- 完整报错和发生时间。
- 近期授权、角色调整或账号变更情况。
- 相关 grant、revoke 或权限检查结果。

## 可能原因
- 用户未被授予目标对象权限。
- 角色未启用或权限未生效。
- 对象属主、schema 或环境选择错误。
- 权限被回收或被策略覆盖。
- 只授予了对象权限，缺少连接、执行或使用权限。

## 标准排查步骤
1. 确认当前用户、目标对象和具体操作。
2. 根据报错判断缺少对象权限、系统权限还是角色权限。
3. 检查用户已授予的角色和权限。
4. 确认对象名、schema 和环境是否正确。
5. 如需授权，先明确最小权限范围和审批要求。
6. 授权后让客户重新登录或重新建立连接验证。

## 常用命令或 SQL 示例
```sql
-- 示例：查看用户权限，具体语法按数据库类型调整
SHOW GRANTS FOR <user>;
SELECT * FROM information_schema.table_privileges WHERE grantee = '<user>';
SELECT * FROM dba_tab_privs WHERE grantee = '<user>';
```

```sql
-- 示例：授权前必须确认最小权限范围
GRANT SELECT ON <schema>.<table> TO <user>;
```

## 风险操作提醒
- 不要索要客户密码或代替客户保存账号凭据。
- 不要建议授予管理员权限作为默认解决方案。
- grant、revoke 可能影响生产访问，必须客户确认范围。
- 涉及账号、角色和权限截图时需要脱敏。

## 需要转人工的情况
- 需要生产环境授权审批。
- 涉及管理员权限、跨系统权限或批量授权。
- 权限变更可能影响审计或合规。
- 客户无法确认对象和最小权限范围。

## 工单摘要要点
- 当前用户、角色和目标对象。
- 执行操作或 SQL 摘要。
- 完整报错摘要。
- 已检查的权限和角色信息。
- 建议的最小授权范围。
- 是否需要审批或人工介入。
