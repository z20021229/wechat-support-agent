# SQL 执行错误

## 适用场景
客户反馈 SQL 执行失败、语法错误、字段不存在、表不存在、对象不存在或数据库返回 ORA、ERROR、syntax 等错误。

## 常见问题现象
- SQL 执行时报语法错误。
- 提示字段不存在、表不存在或对象不存在。
- 同一 SQL 在不同数据库版本表现不同。
- 应用日志中出现 SQL 执行失败。

## 需要向客户收集的信息
- 完整 SQL，敏感值需脱敏。
- 完整报错文本和错误码。
- 数据库类型和版本。
- 表结构、字段名和对象所在 schema。
- SQL 执行入口，例如客户端、应用或任务。
- 最近变更，例如字段调整、版本升级或迁移。

## 可能原因
- SQL 语法不兼容当前数据库类型或版本。
- 表、字段、视图或函数不存在。
- schema、库名或大小写引用错误。
- 参数拼接异常或应用生成 SQL 错误。
- 数据类型不匹配或隐式转换失败。

## 标准排查步骤
1. 获取完整 SQL 和完整报错。
2. 确认数据库类型、版本和执行入口。
3. 检查表、字段、schema 和对象是否存在。
4. 将 SQL 简化到最小复现语句。
5. 对照数据库语法检查关键字、函数和分页写法。
6. 如涉及应用生成 SQL，收集应用侧日志摘要。

## 常用命令或 SQL 示例
```sql
-- 示例：检查对象和字段，具体语法按数据库类型调整
DESCRIBE <table_name>;
SHOW COLUMNS FROM <table_name>;
SELECT column_name FROM information_schema.columns WHERE table_name = '<table_name>';
```

```sql
-- 示例：验证最小查询
SELECT <column_name> FROM <schema>.<table_name> WHERE 1 = 0;
```

## 风险操作提醒
- 不要要求客户提供包含敏感业务数据的完整 SQL 参数。
- 修改表结构、字段或生产 SQL 前必须人工确认。
- 不要直接建议在生产执行未验证的 DDL 或批量 DML。

## 需要转人工的情况
- 涉及生产表结构变更。
- SQL 由核心业务系统自动生成。
- 错误影响批量任务、账务或关键交易。
- 需要数据库专家确认兼容性或执行计划。

## 工单摘要要点
- 数据库类型和版本。
- SQL 摘要和脱敏后的关键片段。
- 错误码和报错摘要。
- 目标表、字段、schema 和对象状态。
- 已完成的最小复现和检查结果。
- 后续需要应用或数据库侧确认的事项。
