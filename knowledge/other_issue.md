# 其他问题

## 适用场景
客户描述的问题暂时无法归入数据库连接、权限、SQL 执行错误、性能或服务不可用分类，需要先补充背景信息。

## 常见问题现象
- 问题描述较模糊，只说异常、失败或不可用。
- 缺少完整报错和操作步骤。
- 涉及多个系统，需要先确认影响范围。
- 无法判断是前端、后端、数据库还是网络问题。

## 需要向客户收集的信息
- 问题背景和业务场景。
- 具体操作步骤和复现方式。
- 完整报错或截图中的脱敏文本。
- 环境信息，例如系统、浏览器、客户端或数据库类型。
- 发生时间、频率和影响范围。
- 最近是否有发布、配置或网络变更。

## 可能原因
- 信息不足，无法判断问题域。
- 操作步骤、环境或输入条件不一致。
- 前端、后端、数据库或网络任一环节异常。
- 最近变更导致兼容性或配置问题。

## 标准排查步骤
1. 引导客户补充背景、步骤、报错和环境。
2. 判断是否能复现，并记录复现条件。
3. 确认影响范围，是单用户、单环境还是全局。
4. 根据补充信息重新归类到具体问题类型。
5. 若仍无法归类，整理已知信息转人工分析。

## 常用命令或 SQL 示例
```bash
curl -I <service-url>
ping <service-host>
tracert <service-host>
```

```sql
-- 示例：确认数据库服务可响应，具体语法按数据库类型调整
SELECT 1;
```

## 风险操作提醒
- 不要要求客户提供密码、密钥、token 或完整未脱敏日志。
- 不要在信息不足时建议删除、重启或修改配置。
- 涉及生产环境操作必须先人工确认。

## 需要转人工的情况
- 客户无法提供足够上下文，但问题影响生产。
- 涉及多个系统协同排查。
- 需要查看生产日志、配置或监控。
- 客户要求执行高风险操作。

## 工单摘要要点
- 问题背景和操作步骤。
- 报错摘要和环境信息。
- 发生时间、频率和影响范围。
- 已收集信息和仍缺失的信息。
- 建议归类或转人工原因。
