# 性能问题

## 适用场景
客户反馈查询慢、系统卡顿、接口响应慢、CPU/内存/IO 占用高、锁等待或任务执行耗时异常。

## 常见问题现象
- 慢 SQL 或查询耗时明显增加。
- 应用页面或接口响应变慢。
- 数据库 CPU、内存或 IO 持续偏高。
- 出现锁等待、连接堆积或超时。

## 需要向客户收集的信息
- 慢 SQL 或慢操作描述。
- 执行耗时、发生时间和频率。
- 数据量、过滤条件和返回行数。
- 执行计划或慢日志摘要。
- CPU、内存、IO 和连接数情况。
- 是否有近期发布、数据增长或配置变更。

## 可能原因
- 缺少索引或执行计划不合理。
- 数据量增长导致原 SQL 不再适配。
- 锁等待、事务未提交或连接池堆积。
- 资源不足，例如 CPU、内存、IO 瓶颈。
- 统计信息过旧或缓存命中率下降。

## 标准排查步骤
1. 确认慢的对象是 SQL、接口、任务还是整体系统。
2. 收集耗时、发生时间、频率和影响范围。
3. 获取慢 SQL、执行计划和数据量信息。
4. 检查 CPU、内存、IO、连接数和锁等待。
5. 对比近期发布、数据量增长和配置变更。
6. 形成低风险建议，例如补充索引评估或错峰执行。

## 常用命令或 SQL 示例
```sql
-- 示例：查看执行计划，具体语法按数据库类型调整
EXPLAIN SELECT * FROM <table_name> WHERE <condition>;
EXPLAIN ANALYZE SELECT * FROM <table_name> WHERE <condition>;
```

```bash
top
vmstat 1 5
iostat -x 1 5
```

## 风险操作提醒
- 不要直接建议生产环境创建或删除索引。
- kill 会话、重启服务、清理缓存属于高风险操作。
- 调整参数可能影响全局性能，必须人工确认。

## 需要转人工的情况
- 涉及生产 SQL 优化、索引变更或参数调整。
- 出现大面积性能下降或业务不可用。
- 需要分析执行计划、锁等待或资源瓶颈。
- 客户要求直接执行高风险操作。

## 工单摘要要点
- 慢 SQL 或慢操作描述。
- 耗时、发生时间、频率和影响范围。
- 数据量、执行计划和资源指标摘要。
- 已确认的锁等待、连接数或近期变更。
- 当前建议和是否需要专家介入。
