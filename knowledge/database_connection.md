# 数据库连接问题

## 适用场景
客户反馈应用或客户端无法连接数据库、连接超时、连接被拒绝、端口不可达或监听异常。

## 常见问题现象
- 提示连接超时、无法连接、connection refused。
- 应用启动时报数据库连接失败。
- 客户端连接同一地址时偶发成功、偶发失败。
- 数据库端口未监听或网络链路不通。

## 需要向客户收集的信息
- 数据库类型和版本。
- 连接地址、端口、库名或服务名。
- 客户端所在环境和访问路径。
- 完整报错文本和出现时间。
- ping、telnet 或 nc 检查结果。
- 数据库监听状态、防火墙、安全组或白名单配置。

## 可能原因
- 连接地址、端口或服务名配置错误。
- 数据库监听未启动或监听地址不正确。
- 网络、DNS、防火墙、安全组或白名单阻断。
- 数据库连接数耗尽或服务端负载过高。
- 客户端驱动版本与数据库版本不兼容。

## 标准排查步骤
1. 确认客户提供的数据库类型、版本、地址和端口。
2. 从客户端环境测试网络连通性。
3. 在数据库服务器侧确认监听进程和端口状态。
4. 检查防火墙、安全组、白名单和访问控制策略。
5. 对照完整报错判断是网络拒绝、认证失败还是服务端异常。
6. 如为偶发问题，补充连接池配置、连接数和时间段信息。

## 常用命令或 SQL 示例
```bash
ping <database-host>
telnet <database-host> <port>
nc -vz <database-host> <port>
netstat -ano | findstr <port>
```

```sql
-- 示例：查看当前连接信息，具体语法按数据库类型调整
SELECT * FROM v$session;
SHOW PROCESSLIST;
```

## 风险操作提醒
- 不要要求客户提供数据库密码或明文连接串。
- 修改防火墙、安全组或监听配置前需要客户确认影响范围。
- 重启数据库监听或数据库服务属于高风险操作，必须人工确认。

## 需要转人工的情况
- 需要修改生产网络策略、数据库监听或安全组。
- 涉及数据库服务重启。
- 客户无法提供脱敏报错，且问题影响生产业务。
- 疑似数据库服务端故障或资源耗尽。

## 工单摘要要点
- 数据库类型、版本、连接地址和端口。
- 客户端环境和访问路径。
- 完整报错摘要。
- 连通性检查结果。
- 已确认的网络、安全策略或监听状态。
- 当前影响范围和是否需要人工介入。
